# Domain Restraint Doctrine

Energy Theory is a magpie, not a mimic.

It may take the shiny shape of a question from any domain.

It may not surrender its root to any domain.

## Rule

```text
Domains influence tools.
Domains do not alter the root.
```

## Root-protection test

A proposed term, formula, metaphor, or example may enter METAPAT only if it passes all of these checks:

1. It clarifies a root or derived term.
2. It does not require its source domain to be true everywhere.
3. It does not replace Energy Theory vocabulary with domain vocabulary.
4. It states what transfers and what does not.
5. It remains demotable if it overreaches.

## Domain capture failures

Reject or demote a proposal when it does any of the following:

- Makes conservation law root because physics uses conservation law.
- Makes consciousness root because psychology or phenomenology uses consciousness.
- Makes wave mechanics root because scalar/vector can be explicated as standing/propagating waves.
- Makes algebraic tensors own the primitive because tensor is a mathematical word.
- Makes language own questions because questions are linguistic in ordinary use.
- Makes code logs own registration because log files can register.

## Correct import pattern

Use this form:

```text
Domain term: <term>
Status: explicationary tool / example / resonance / application constraint
Clarifies: <Energy Theory term>
Transfers: <what carries>
Does not transfer: <what remains domain-specific>
Root impact: none
```

## Example: wave language

```text
Domain term: standing wave / propagating wave
Status: explicationary tool
Clarifies: scalar / vector
Transfers: held versus motioned energy-state behavior
Does not transfer: full wave-physics ontology, conservation assumptions, medium-specific equations
Root impact: none
```

## Example: consciousness

```text
Domain term: consciousness
Status: observer-mode / registration application
Clarifies: story, cognitive gestalt, mind-instantiation
Transfers: conscious narration of registered time
Does not transfer: requirement that time, registration, legibility, readability, or object-instantiation require mind
Root impact: none
```

## Example: cup

```text
Domain term: cup
Status: example
Clarifies: formed object as tensor of simplexes
Transfers: boundary-simplex + space-within simplex + integration at scale
Does not transfer: all objecthood reduces to cups, containers, or human use
Root impact: none
```
